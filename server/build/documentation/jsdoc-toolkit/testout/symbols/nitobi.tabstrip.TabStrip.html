Class nitobi.tabstrip.TabStrip


Extends nitobi.ui.Element.
A nitobi tabstrip. A tabstrip declaration takes this form: 


<ntb:tabstrip id="SimpleTabstrip" width="800px" height="600px">
	<ntb:tabs height="" align="center" overlap="15">
		<ntb:tab width="190px" tooltip="Welcome." label="IFrame Tab" source="http://www.nitobi.com" containertype="iframe"></ntb:tab>
		<ntb:tab width="190px" tooltip="Welcome." label="DOM Tab" source="tab2"></ntb:tab>
		<ntb:tab width="190px" tooltip="Welcome." label="Ajax Tab" source="tab3.html" loadondemandenabled="true"></ntb:tab>
	</ntb:tabs>
</ntb:tabstrip>
<div id="tab2">
	<h1>DOM Tab</h1>
	<img src="images/nitobi.jpg" />
</div>
Calling nitobi.loadComponent('SimpleTabstrip'); will render a tabstrip at the location of that XML declaration in your web page. To access this API you get the tabstrip object from its node in the HTML document. We can, for example, change the source of the first tab: 

var tabstrip = $('tabstrip1').jsObject;
var tab = tabstrip.getTabs().get(0);
tab.setSource('http://www.google.com');
Note: some operations require that you call nitobi.tabstrip.TabStrip#render after changing some properties. This is due to the expense of changing them and rendering them individually.
Defined in: tabstrip.js.

Example

{
	var tabstrip = nitobi.getComponent('tabstrip1');
	var tabs = tabstrip.getTabs();
	var tab = tabs.get(0);
	tab.setLabel("A new label for the 1st tab");
}

render()
Renders the tabstrip. The first time the tabstrip is rendered, both the tabs and their body containers are rendered. On subsequent render calls, unless tabs are added and removed, only the tabs are rendered. This prevents the user losing data in a tab body.
